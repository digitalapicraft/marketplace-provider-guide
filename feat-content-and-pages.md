Manage the static storefront: standing pages (About, Help, Privacy, Terms, homepage), articles, and the media library that holds every image, PDF, and file the portal serves. Edit content when copy is out of date or when you launch a new page.

![Figure. The Pages list showing every standing page with status, author, and last-changed columns.](.gitbook/assets/screenshots/provider/admin-content-pages.png)

## Configure

1. Expand **Content** in the sidebar, then click **Pages** (or **Media**).
2. To edit a page, find it with the **Title** filter, click the title, then click **Edit**.
3. To add a page, click **Add page**, then enter a **Title**, a short **URL alias** like `/partner-programme`, and a **Description** for search engines.
4. Build the body with the visual block editor, dragging heading, text, image, and callout blocks onto the canvas.
5. To add an image or file, open **Media**, click **Add media**, upload the file, set a **Name** and alt text, then **Save**. Pages reference media items, so replacing a file updates every page that uses it.
6. To gate a change behind review, set **Status** to *Unpublished*, save, preview, then re-publish once approved.
7. To schedule a release, open the **Scheduling** panel, set a **Publish on** date, and save as *Draft*. The marketplace publishes it at that time.
8. Click **Save** to commit, **Preview** to render the page as a visitor sees it.

{% hint style="success" %}
**Result:** The page or article is live at its alias, and referenced media serves the current file. Published changes appear on the next page load.
**Tip:** Paste copy through a plain-text editor first. Pasting straight from a word processor drags styling that breaks the layout.
{% endhint %}

### Key controls

1. **Title / Status filters.** Find any page and scope to Published or Unpublished rows.
2. **Add page.** Opens the new-page editor.
3. **Operations menu.** Per-row Edit, revision log, and remove actions.
